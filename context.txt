한국어도 답변
터미널 호출 권한이 필요한 사항은 직접해결하지 말고 요청
이외 직접 수정가능한 코드는 직접 수행

백엔드 (Python, Flask)
프레임워크: Flask
진입점: backend/app.py
데이터베이스: SQLite (backend/database.db 파일)
daily_logs (일일 업무 기록), daily_tasks (세부 업무 내용), file_history (업로드 파일 이력) 등의 테이블을 사용합니다.
주요 기능 및 API:
전통적인 웹 서버처럼 HTML 템플릿을 렌더링하는 라우트('/', '/work/daily' 등)가 다수 존재합니다.
React 프론트엔드와 통신하기 위한 REST API가 구현되어 있습니다.
GET /api/work/daily: 전체 일일 업무 기록을 JSON 형태로 제공합니다.
GET /api/test: API 동작 테스트용 엔드포인트입니다.
.evtx 같은 파일을 업로드하고 관리하는 기능이 있습니다.
특이사항:
flask-cors가 설정되어 있어, 다른 포트에서 실행되는 프론트엔드(예: React 개발 서버)의 API 요청을 허용합니다.
if __name__ == '__main__': 구문을 통해 host='0.0.0.0', port=5000으로 서버가 실행됩니다.
프론트엔드 (JavaScript, React)
프레임워크/라이브러리: React 19
빌드 도구: Vite (정확히는 rolldown-vite)
주요 파일:
진입점: frontend/src/main.jsx
메인 컴포넌트: frontend/src/App.jsx
스타일링: Tailwind CSS
주요 기능:
App.jsx 컴포넌트는 렌더링 시 useEffect를 사용해 백엔드 API(http://127.0.0.1:5000/api/work/daily)를 호출합니다.
API 호출을 통해 받아온 데이터(work_date)를 화면에 리스트 형태로 표시합니다.
데이터를 기다리는 동안 "Loading...", 오류 발생 시 에러 메시지를 표시하는 기능이 구현되어 있습니다.
주요 의존성:
react, react-dom
lucide-react (아이콘 라이브러리)
tailwindcss (CSS 프레임워크)
vite (빌드 및 개발 서버)
